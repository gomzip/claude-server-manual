# 서버 관리자를 위한 Claude Code 매뉴얼 작성 최적 프롬프트

> 아래 프롬프트를 AI(Claude 등)에 입력하면 RHEL 8/9 기반 서버 관리 매뉴얼이 생성됩니다.
> 필요에 따라 [ ] 안의 내용을 수정하여 사용하세요.

Microsoft Word(.docx) 문서로 작성하세요.

### 2.2 사전 패키지 설치 (터미널 화면으로)
순서:
1. 현재 OS 버전 확인: cat /etc/redhat-release
2. EPEL 저장소 활성화
3. Node.js 20.x LTS 설치 (NodeSource 공식 저장소)
   - RHEL 8: yum module reset nodejs && yum module enable nodejs:20
   - RHEL 9: dnf module reset nodejs && dnf module enable nodejs:20
4. npm 최신 버전 업데이트
5. 기타 필수 패키지 (curl, git, jq)
6. 설치 확인 (node -v, npm -v)

각 단계마다 터미널 화면 형식으로 명령어 + 예상 출력을 보여줄 것

### 2.3 Claude Code 설치
- npm install -g @anthropic-ai/claude-code
- 설치 확인: claude --version (출력 화면 포함)
- PATH 설정이 필요한 경우 안내
- 방화벽 설정: firewall-cmd로 필요한 포트 오픈 (필요 시)

### 3.1 최초 실행
- claude 명령어 실행 시 나타나는 초기 화면 (터미널 형식)
- 주요 실행 옵션 표:
  | 옵션 | 설명 | 사용 예시 |
  |------|------|-----------|
  | (없음) | 대화형 모드 | claude |
  | -p "..." | 비대화형 실행 | claude -p "서버 상태 점검" |
  | --model | 모델 지정 | claude --model opus |
  | --output-format | 출력 형식 | claude -p "..." --output-format json |
  | --verbose | 상세 출력 | claude --verbose |
  | --allowedTools | 허용 도구 | claude --allowedTools "Bash,Read" |
  | --debug | 디버그 모드 | claude --debug |
  * 서버 관리에 자주 쓰이는 옵션 위주로 10개 이상 정리

### 3.2 OAuth 인증 (Claude Max 구독 사용자)
단계별 설명:
1. claude 최초 실행 → 인증 URL 표시 화면
2. 브라우저가 있는 환경: 자동 열림 → 로그인 → 승인
3. ⚠️ 브라우저가 없는 서버(SSH) 환경:
   - 표시된 URL을 로컬 PC 브라우저에 복사하여 인증
   - 인증 코드 복사 → 터미널에 붙여넣기
   - 전체 과정을 터미널 화면으로 표현
4. 인증 토큰 저장 위치: ~/.claude/ 디렉토리
5. 토큰 갱신 방법 및 만료 시 재인증 안내

### 3.3 API Key 인증
1. Anthropic Console(console.anthropic.com)에서 API Key 발급 방법 (간략)
2. 환경변수 설정:
   export ANTHROPIC_API_KEY="sk-ant-xxxxx..."
3. 영구 등록:
   - ~/.bashrc에 추가
   - 또는 /etc/environment에 시스템 전체 적용
4. 보안 주의사항:
   - chmod 600 ~/.bashrc
   - API Key를 스크립트에 하드코딩 금지
   - /etc/profile.d/claude-env.sh 파일로 분리 관리 권장

4장. 서버 관리 프롬프트 샘플 (25~35 페이지)
========================================
⚡ 이 장이 매뉴얼의 핵심입니다.

**각 프롬프트마다 반드시 아래 5가지를 모두 포함할 것:**
① 프롬프트(명령어) 전문 — 복사해서 바로 쓸 수 있게
② 실행 방법 — 대화형(claude 진입 후 입력) / 비대화형(claude -p "...")
③ 예상 출력 화면 — 터미널 박스 형식으로 상세하게
④ 출력 결과 해석 방법 — 어떤 값이 정상이고 비정상인지
⑤ 후속 조치 가이드 — 문제 발견 시 어떻게 대응할지

### 4.1 시스템 상태 점검 (4개 이상 프롬프트)
- 서버 전체 헬스체크:
  "현재 서버의 전체적인 상태를 점검해줘. CPU, 메모리, 디스크,
   네트워크, 실행 중인 서비스 상태를 종합적으로 분석하고
   이상 여부를 알려줘."
- 디스크 사용량 분석 및 정리 추천
- 메모리/CPU 사용률 분석 및 이상 프로세스 탐지
- 좀비 프로세스 및 이상 프로세스 탐지

### 4.2 로그 분석 (5개 이상 프롬프트)
- /var/log/messages 최근 24시간 분석
- /var/log/secure 로그인 시도 분석 (무차별 대입 탐지)
- journalctl 기반 실패 서비스 분석
- 특정 기간 로그 필터링 및 요약 (날짜 범위 지정)
- 웹서버(httpd/nginx) 액세스/에러 로그 분석

### 4.3 보안 취약점 분석 (7개 이상 프롬프트)
- 열린 포트 및 리스닝 서비스 점검
- SSH 설정(/etc/ssh/sshd_config) 보안 감사
- 파일 권한 이상 탐지 (SUID/SGID 비트, world-writable 파일)
- 사용자 계정 보안 점검 (비밀번호 정책, 미사용 계정, UID 0 계정)
- SELinux 상태 및 정책 위반 점검
- 최근 보안 패치 적용 여부 확인 (yum/dnf updateinfo)
- firewalld 규칙 검토 및 불필요한 서비스 노출 점검

### 4.4 서버 설정 최적화 (4개 이상 프롬프트)
- 커널 파라미터(sysctl.conf) 최적화 분석
- systemd 서비스 시작/중지/활성화 상태 점검
- 네트워크 설정(bonding, VLAN, DNS 등) 검토
- NTP/Chrony 시간 동기화 점검

### 4.5 장애 대응 (4개 이상 프롬프트)
- 특정 서비스 다운 원인 분석
- 디스크 풀(100%) 긴급 대응 (큰 파일/로그 탐지 및 정리 안내)
- 높은 로드(Load Average) 원인 분석
- OOM Killer 발동 원인 분석 및 방지 설정

5장. 자동화 스케줄링 (10~15 페이지)

### 5.1 비대화형 실행 기본

### 5.2 스케줄 추천 및 cron 적용

아래 표로 정리:
| 주기 | 점검 항목 | 중요도 | 예상 소요시간 |
|------|-----------|--------|--------------|
| 매시간 | 디스크 사용률 80% 초과 체크 | 🔴 높음 | ~1분 |
| 매시간 | 서비스 상태(httpd, sshd 등) 체크 | 🔴 높음 | ~1분 |
| 매시간 | 최근 1시간 에러 로그 건수 | 🔴 높음 | ~2분 |
| 매일 06:00 | 보안 로그(/var/log/secure) 분석 | 🔴 높음 | ~3분 |
| 매일 07:00 | 시스템 리소스 24시간 트렌드 | 🟡 중간 | ~3분 |
| 매일 22:00 | 백업 상태 확인 | 🔴 높음 | ~2분 |
| 매주 월 09:00 | 보안 취약점 종합 점검 | 🟡 중간 | ~5분 |
| 매주 월 09:30 | 패치 현황 및 업데이트 가능 목록 | 🟡 중간 | ~3분 |
| 매주 금 17:00 | 사용자 계정 감사 | 🟡 중간 | ~2분 |
| 매월 1일 10:00 | 서버 설정 종합 리뷰 | 🟢 낮음 | ~10분 |
| 매월 1일 11:00 | 성능 최적화 분석 | 🟢 낮음 | ~5분 |
| 매월 1일 14:00 | 용량 계획(3개월 추세 분석) | 🟢 낮음 | ~5분 |

**각 주기별로 다음을 제공:**
1. 실행 스크립트 전문 (.sh 파일) — /opt/claude-scripts/ 에 저장
2. crontab -e 등록 내용 (정확한 cron 표현식)
3. 로그 저장 경로 및 logrotate 설정
4. 실행 결과 예시 화면 (터미널 박스)

**매시간 점검 스크립트 예시:**
/opt/claude-scripts/hourly-check.sh 전체 코드를 보여줄 것
- 디스크, 서비스, 에러 로그를 한번에 점검
- 이상 발견 시 알림 함수 호출
- 결과를 /var/log/claude/hourly/ 에 날짜별 저장

**매일 점검 스크립트 예시:**
/opt/claude-scripts/daily-security.sh 전체 코드를 보여줄 것
/opt/claude-scripts/daily-resource.sh 전체 코드를 보여줄 것

**매주 점검 스크립트 예시:**
/opt/claude-scripts/weekly-audit.sh 전체 코드를 보여줄 것

### 5.3 스크립트 통합 관리
- /opt/claude-scripts/ 디렉토리 구조:
  /opt/claude-scripts/
  ├── hourly-check.sh
  ├── daily-security.sh
  ├── daily-resource.sh
  ├── weekly-audit.sh
  ├── monthly-review.sh

6장. 실시간 모니터링 (8~10 페이지)
========================================

### 6.1 실시간 모니터링 개념
- Claude Code 모니터링 vs 기존 도구(Zabbix, Prometheus, Grafana) 비교표:
  | 비교 항목 | Claude Code | 기존 모니터링 도구 |
  |-----------|-------------|-------------------|
  | 설치 복잡도 | ... | ... |
  | 분석 능력 | ... | ... |
  | 실시간성 | ... | ... |
  | 비용 | ... | ... |
  | 적합한 용도 | ... | ... |
- Claude Code는 "분석"에, 기존 도구는 "수집·시각화"에 강점 → 상호 보완 권장

### 6.2 실시간 로그 감시 스크립트
- tail -F /var/log/messages | 패턴 감지 → claude -p 파이프라인 구성
- 전체 스크립트 코드 제공
- systemd 서비스 유닛 파일로 등록하는 방법:
  /etc/systemd/system/claude-log-monitor.service 전문
  systemctl enable/start 명령어
- 이상 탐지 시 자동으로 알림 트리거하는 방법

### 6.3 임계값 기반 모니터링
- CPU 사용률 90% 초과, 메모리 85% 초과, 디스크 90% 초과 시
- 임계값 감지 → Claude Code로 원인 자동 분석 → 알림 전송
- 전체 파이프라인 스크립트 제공
- 실행 결과 예시 화면

### 6.4 모니터링 종합 대시보드 스크립트
- 주요 지표(CPU, MEM, DISK, LOAD, 서비스 상태, 최근 에러)를
  한 화면에 보여주는 bash 스크립트
- 터미널 대시보드 출력 예시 화면:

### 7.1 텔레그램 (Telegram) 연동
1. 텔레그램 봇 생성 (BotFather 대화 화면 예시 포함):
   - /newbot 명령어 → 봇 이름/유저네임 설정 → 토큰 수령

3. curl을 이용한 메시지 전송 함수:
   send_telegram() 함수 코드 전문
4. Claude Code 분석 결과 → 텔레그램 전송 통합 스크립트
5. 실제 수신 화면 예시 (텍스트 형태로 메시지 박스 표현)

### 7.2 슬랙 (Slack) 연동
1. Slack App 생성 및 Incoming Webhook URL 발급 방법
2. curl을 이용한 메시지 전송 함수:
   send_slack() 함수 코드 전문
3. Slack 서식(Block Kit) 활용 — 경고 레벨별 색상 구분
4. Claude Code 분석 결과 → 슬랙 전송 통합 스크립트
5. 실제 수신 화면 예시

### 7.3 디스코드 (Discord) 연동
1. Discord 서버에 Webhook URL 생성 방법
2. curl을 이용한 메시지 전송 함수:
   send_discord() 함수 코드 전문
3. Embed 형식 활용 — 구조화된 알림 메시지
4. Claude Code 분석 결과 → 디스코드 전송 통합 스크립트
5. 실제 수신 화면 예시

### 7.4 알림 통합 관리
- /opt/claude-scripts/lib/send_alert.sh 공통 함수 전문:
  - send_alert(level, title, message) 형태
  - level: INFO / WARNING / CRITICAL
  - config.env에서 활성화된 플랫폼으로 자동 전송
- 알림 레벨별 동작 정의표:
  | 레벨 | 텔레그램 | 슬랙 | 디스코드 | 설명 |
- 중복 알림 방지 로직 (동일 알림 10분 이내 재발송 차단)
- config.env 설정 파일 전문:
  ALERT_TELEGRAM_ENABLED=true
  ALERT_TELEGRAM_BOT_TOKEN=xxxx
  ALERT_TELEGRAM_CHAT_ID=xxxx
  ALERT_SLACK_ENABLED=false
  ...

| 응답이 너무 느림 | 모델/네트워크 문제 | --model 변경, 프록시 설정 확인 |

## 사용 가이드

### 이 프롬프트를 사용하는 방법

1. **Claude에서 직접 사용**: 위 프롬프트 전체를 Claude에 입력
2. **분할 사용**: 너무 길면 장(chapter) 단위로 나눠서 요청
   - "위 프롬프트의 1~3장을 먼저 작성해줘"
   - "이어서 4장을 작성해줘"
   - "이어서 5~6장을 작성해줘"
   - ...
3. **커스터마이징**: [ ] 안의 값이나 표의 내용을 환경에 맞게 수정
